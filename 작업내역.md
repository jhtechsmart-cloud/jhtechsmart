## 5차 작업 — 평균매출 판정 로직 수정

**기준:** 소공인 스마트제조 지원사업 평균매출 조건은 **2억원 초과** 기업이 지원 대상

| 조건 | 변경 전 | 변경 후 |
|---|---|---|
| 평균매출 > 2억원 | 초록 "기준 충족 가능" | 초록 **"2억원 초과 — 지원 대상 가능"** |
| 평균매출 ≤ 2억원 | 빨강 "기준 초과 가능성 있음" | 빨강 **"2억원 이하 — 기준 미달 (지원 대상 제외)"** |

판정 로직: `avg <= limit` (이하면 OK) → `avg > limit` (초과해야 OK) 로 수정

### 12-B. 견적번호 형식 변경

`generateQuoteNo()` 산출물: `YYMMDD-NNNN` → **`YYMMDD-NNNN-v1`** (v1 접미사 부착).
`getNextVersion()`은 `-v1` → `-v2` → `-v3` ... 증분. 레거시 `-R\d+`도 다음 버전부터 `-v` 형식으로 자동 전환.

### 12-C. 담당자 = 시트 단일 진실 소스

- `DEFAULT_USERS` 하드코딩 배열 **삭제**. `getUsers()` 폴백은 빈 배열
- `u.title` 참조 **8군데 모두 제거** (`u.name + ' ' + u.title` → `u.name`)
- 신규 담당자 등록 폼: 이름란에 "이름 + 직책" 함께 입력하도록 안내
- 직책 컬럼이 이미 사용자에 의해 시트에서 삭제됨 → 코드 인덱스도 6컬럼으로 시프트

### 12-E. 세션 자동 로그인

새로고침 시 로그아웃 방지 — `sessionStorage('jhtech_session_user')`에 사용자 ID만 저장.
탭 종료 시 자동 정리되어 공용 PC에서도 비교적 안전. 비밀번호는 저장하지 않음.

```
doLogin → sessionStorage.setItem → _enterApp
logout → sessionStorage.removeItem
initApp 끝 → tryRestoreSession (담당자 시트 로드 후 사용자 객체 복원)
```

- 신규 컬럼은 무조건 `UNIFIED_HEADERS` 배열 끝에 push

### Notion 속성 타입 일치
- 노션에서 속성 타입을 바꾸면 `NOTION_PROP_MAP`도 같이 바꿔야 함 (안 그러면 validation_error)
- `ensureNotionSchema()`가 mismatch 경고 로그 — 운영 후 한 번씩 실행 권장

### 시트→노션 push 직후 hash 저장 필수
- `pushToNotion` 끝에서 hash를 저장하지 않으면 sync가 push 결과를 "새 노션 수정"으로 오인 → 무한루프
- 코드에 이미 적용되어 있음 (line 1006~)

### SpreadsheetApp.flush() 타이밍
- `setValues` 직후 `_readUnifiedRow`를 호출하면 옛 값을 읽을 수 있음 (batch write 미반영)
- `_safePushToNotion`이 push 전 `SpreadsheetApp.flush()` 호출하도록 보강됨

### OpenAI max_tokens
- 명시 안 하면 짧게 잘림 (PART 2 마커에서 truncation 발생 사례)
- `max_tokens: 2500` + `finish_reason === 'stop'` 검증 둘 다 필요

### Mailer Web App 계정 분리
- `MailApp.sendEmail` 발신자는 GAS 소유자 계정
- 메인 GAS = `smart@paxc.co.kr` / 발신은 `jhtechsmart@gmail.com` → Mailer 별도 배포 필수
- 두 Script Properties (`MAILER_WEBAPP_URL`, `MAILER_TOKEN`)을 메인 + Mailer 양쪽에 정확히 동일하게

### 사전 등록 페이지 매칭 키
- 사업자번호 매칭이 동작하려면 사전 등록 페이지에 **사업자번호 필수**
- 같은 사업자번호로 사전 페이지 2개 이상이면 첫 매칭만 사용 (나머지 무시)
- archive(휴지통) 상태 페이지는 query 결과에서 제외 — 새 페이지 생성됨
